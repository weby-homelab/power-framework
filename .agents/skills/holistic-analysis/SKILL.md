# 🛡️ Holistic Analysis & Step-by-Step Verification Protocol

Цей скілл вимагає від агента дотримуватись суворого циклу аналізу та впровадження змін у будь-якому проекті.

## 📋 Інструкція з виконання:

1. **Етап 1: Послідовний Аналіз (Sequential Analysis First)**
   * ЗАБОРОНЕНО писати код або пропонувати рішення без повного вивчення контексту.
   * Знайди та прочитай всі пов'язані файли проекту за допомогою `grep_search` або `view_file` (зчитуй по черзі, а не великими пачками).
   * Задокументуй виявлені архітектурні баги, застарілі патерни, зайві дискові/мережеві операції (I/O) та небезпечні регулярні вирази у списку по порядку.

2. **Етап 2: Покрокове Впровадження (Step-by-Step Act)**
   * Розбий роботу на мінімальні ізольовані кроки.
   * Впроваджуй лише одну зміну за один раз (наприклад, один `replace_file_content` на файл за один крок).

3. **Етап 3: Негайний Зворотний Зв'язок та Тестування (Validate)**
   * Після кожної найменшої зміни запусти перевірку синтаксису (`python -m py_compile` чи відповідні лінтери/тести).
   * Не переходь до наступного кроку, доки поточний крок не завершиться успішною валідацією з нульовою кількістю помилок.

4. **Етап 4: Глобальний Аудит (Final Quality Check)**
   * Перевір за чек-листом:
     * Чи типізовані та валідовані всі вхідні дані на рівні API (Pydantic / stdlib)?
     * Чи оптимізовано I/O (немає зайвих дискових операцій на швидких шляхах)?
     * Чи обмежено створення фонових потоків (використовується `ThreadPoolExecutor` замість сирих потоків)?
     * Чи очищено образ/кодову базу (видалено systemd, пусті каталоги)?
