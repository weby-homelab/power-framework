## 🧠 5 "Супер фішок" P.O.W.E.R. 3.2.1

### 1. ⚡ Економія контексту до 95% (Гроші й токени залишаються у вас)
Замість того, щоб змушувати AI-агента вичитувати сотні файлів (що коштує до $2-5 за одну сесію у багатьох моделей), POWER надає:
- **Ієрархічні Sub-indexes (`_index.md`)**: Агент бачить карту всієї бази у стислому вигляді (1-2 КБ токенів).
- **Точковий пошук Chunking**: Повертаються лише релевантні фрагменти з точним вказуванням файлу та рядків.

### 2. 🛡️ Безпека пам'яті: Працює навіть на 8–12 GB RAM VPS та LXC
Більшість GraphRAG фреймворків тягнуть за собою PyTorch, CUDA-залежності та вантажать гігабайти вагових коефіцієнтів, викликаючи фатальні `Out Of Memory (OOM)` краші.  
POWER 3.2.1 працює на **прямому ONNX Runtime (`BGEM3OnnxManager`)**:
- Провайдер **`bge-m3`** (`aapot/bge-m3-onnx`, 1024d) завантажується напряму через C++ біндінги.
- Адаптивне зменшення батчів та приборканий алокатор пам'яті BFCArena.
- Споживання RAM фіксується у межах **< 1.8 ГБ**!

### 3. 🌐 Двомовна суперсила (UA ↔ EN)
Завдяки інтеграції моделі **BGE-M3 (1024-dimensional dense vectors)**, агент легко знаходить нотатки:
- Запит українською: `"як налаштувати оркестрацію деплою"`
- Знаходить документ англійською: `01_Projects/Docker_Compose_Production_Setup.md` з точністю **95%+**!

### 4. 🎯 Канонічний 3-рівневий Пошук (`reranked`)
POWER 3.2.1 комбінує найкраще з усіх світів через **Reciprocal Rank Fusion (RRF)**:
1. **SQLite FTS5 (BM25)** — шукає точні ключові слова, змінні, класи, функції.
2. **BGE-M3 Dense Vector** — знаходить глибокий концептуальний зміст.
3. **BGE Reranker v2 M3 (Cross-Encoder)** — переранжує топ-результати з урахуванням контексту.

### 5. 🩺 Авто-лікування та Захист від битих посилань
- **`power lint`**: Валідує структуру метаданих OKF, знаходить биті вікіпосилання `[[Note Name]]` та сирі нотатки без заголовків.
- **`power heal`**: Автоматично відновлює поламані посилання та генерує відсутні теги й описи через LLM.
- **`power rot`**: Знаходить застарілі, дубльовані та суперечливі нотатки.

## ⚙️ Як розпочати за 2 хвилини?

### 1. Встановлення
```bash
pip install git+https://github.com/weby-homelab/power-framework.git
```

### 2. Ініціалізація вашої бази знань (P.A.R.A. + OKF)
```bash
power init /шлях/до/вашого/second-brain
```

### 3. Підключення до вашого AI-агента (MCP Config)
Додайте у конфіг вашого агента (`opencode.jsonc`, `cline_config.json`, Cursor/Windsurf):

```json
{
  "mcpServers": {
    "power": {
      "command": "power-mcp",
      "env": {
        "POWER_VAULT_PATH": "/absolute/path/to/second-brain",
        "POWER_EMBED_PROVIDER": "bge-m3"
      }
    }
  }
}
```

Всі 12 MCP-інструментів (`ingest_note`, `search_vault_tool`, `lint_vault`, `generate_index`, `read_sub_index`, `heal_frontmatter_tool` тощо) стануть доступні вашому AI-агенту миттєво!

## ❓ Часте питання: Чи не обмежує методологія P.A.R.A. гнучкість використання?

**Коротка відповідь: Ні, взагалі не обмежує! P.A.R.A. — це опціональна зручність, а не примус.**

- **Повна свобода структури папок**: Ви вільні зберігати файли у будь-яких власних папках (`my_docs/`, `recipes/`, `ideas/`, `code/` або в єдиному корені). Фреймворк проіндексує та знайде їх усюди.
- **Тип визначається метаданими**: Категорія нотатки зазначається прямо в YAML-шапці файлу (`type: Resource` або `type: Project`).
- **Навіщо префікси P.A.R.A. (`01_Projects/` і т.д.)?**: Вони потрібні лише для того, щоб категорія підхоплювалася **автоматично**, якщо людина чи AI-агент забули вказати `type` у метаданих.

P.O.W.E.R. 3.2.1 працює з будь-якою існуючою структурою вашого Obsidian або звичайних Markdown-файлів без жодних обмежень!
